       - Await params to get id (Next.js 15 async params pattern)
       - Create Supabase client, query invoice by id with select('*, loads(load_number, pickup_company, delivery_company)')

       - Optionally store PDF in Supabase Storage ('invoice-documents' bucket, path: `${invoice.org_id}/${invoice.id}.pdf`) and update invoice.pdf_url
       - Wrap in try/catch, return 500 on renderToBuffer failure

    Note: Do NOT use server actions for this -- server actions cannot return binary data. Route handlers are the correct pattern per RESEARCH.
