    1. Create `src/hooks/use-realtime-dashboard.ts`:

       - Use useRef for Supabase client, router.refresh() on any change
       - Export useRealtimeDashboard(orgId: string | null)

    2. Create `src/app/(app)/dashboard/stat-cards.tsx`:
       - Client component receiving stat values as props
       - Props: { activeLoads: number; bookedToday: number; driversOnDuty: number; revenueMtd: number }
       - Render 4 cards in a grid (4 cols on lg, 2 on md, 1 on sm)
       - Each card: icon (from lucide-react), label, value (large bold), subtle background color
       - Icons: Truck for active loads, Package for booked today, Users for drivers on duty, DollarSign for revenue MTD
       - Revenue MTD formatted with Intl.NumberFormat as USD currency

    3. Create `src/app/(app)/dashboard/activity-feed.tsx`:
       - Client component receiving merged activity items as props
       - Props: { items: ActivityItem[] } where ActivityItem = { id, type ('status_change'|'dispatch'|'invoice'), description, timestamp, link? }
       - Render as a chronological list with relative timestamps (using date-fns formatDistanceToNow)
       - Each item: icon by type, description text, time ago
       - Max height with scroll, or show first 20 items

    4. Create `src/app/(app)/dashboard/quick-actions.tsx`:
       - Client component with 3 action buttons
       - "Create Load" -> /loads/new (Plus icon)
       - "Dispatch Driver" -> /dispatch (Truck icon)
       - "Create Invoice" -> /invoices/new (FileText icon)
       - Styled as outlined/secondary buttons in a horizontal row

    5. Create `src/app/(app)/dashboard/dashboard-view.tsx`:
       - Client wrapper that calls useRealtimeDashboard(orgId) for live updates
       - Renders StatCards, ActivityFeed, and QuickActions
       - Receives all data as props from server component

    6. Replace `src/app/(app)/dashboard/page.tsx`:
       - Server component
       - Detect if user is Owner-Operator: query profiles for user's org_id, then count org_members. If count === 1 and role === 'admin', treat as Owner-Operator. If Owner-Operator, query for user's linked driver record.
       - Run 4 stat queries in parallel via Promise.all:
         - Active loads: count where status = 'in_transit' (Owner-Op: filter by driver_id)
         - Booked today: count where created_at >= startOfToday (Owner-Op: filter by driver_id)
         - Drivers on duty: count where status = 'active' AND current_vehicle_id is not null (Owner-Op: just show 1 or 0 based on own status)
         - Revenue MTD: sum total_charges where status IN ('delivered','invoiced','paid') AND delivery_date >= first of current month (Owner-Op: filter by driver_id)
       - Run activity feed queries in parallel:
         - Last 10 load_status_history with load_number join (Owner-Op: filter by load's driver_id)
         - Last 5 dispatches with load_number and driver name join (Owner-Op: filter by driver_id)
         - Last 5 invoices (Owner-Op: filter by load's driver_id)
       - Merge activity items, sort by created_at descending, format into ActivityItem[]
       - Pass all data to DashboardView
       - Owner-Operator view: show "Your Operation" title instead of "Dashboard", show single vehicle status card if driver has current_vehicle_id
  </action>
