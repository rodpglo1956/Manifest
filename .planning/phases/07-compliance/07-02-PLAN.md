  <files>supabase/migrations/00022_compliance_scanner.sql</files>
  <action>
Create `supabase/migrations/00022_compliance_scanner.sql` following the exact pattern from 00019_alert_generators.sql:

1. `check_compliance_items()` function (security definer, set search_path = ''):
   - Scan all compliance_items where status NOT IN ('completed', 'waived', 'not_applicable') and due_date IS NOT NULL
   - Join compliance_profiles for org_id
   - Calculate days_until_due = due_date - current_date
   - Update item status: overdue if days < 0, due_soon if days <= 30
   - Generate compliance_alerts at threshold days (check against alert_days_before array using `= any()`)
   - De-duplicate alerts with NOT EXISTS check within 24-hour window
   - Determine alert_type: 'overdue' (< 0 days), 'due_soon' (<= 7 days), 'approaching' (> 7 days)
   - Determine severity: 'critical' (overdue), 'warning' (due_soon), 'info' (approaching)
   - Also INSERT into proactive_alerts with alert_type = 'compliance_' || _alert_type for Marie visibility
   - Set related_entity_type = 'compliance_item', related_entity_id = item.id

2. Auto-generate next occurrence for completed recurring items:
   - Select completed items with recurrence_rule IS NOT NULL
   - Use NOT EXISTS to avoid duplicates (check for same category + vehicle_id + driver_id + due_date > current item)
   - Calculate next due_date using PostgreSQL interval arithmetic:
     - annual: + interval '1 year'
     - biennial: + interval '2 years'
     - quarterly: + interval '3 months'
     - monthly: + interval '1 month'
     - custom: + (recurrence_months || ' months')::interval

3. Schedule with pg_cron: `SELECT cron.schedule('check-compliance-items', '0 6 * * *', $$SELECT check_compliance_items();$$);`

Use `create extension if not exists pg_cron;` at top if not already created (check 00019 pattern).

<task type="auto">
  <name>Task 2: Compliance server actions for all CRUD operations</name>
  <files>src/lib/compliance/actions.ts</files>
  <action>
Create `src/lib/compliance/actions.ts` with 'use server' directive, following patterns from src/lib/alerts/actions.ts:

**Profile actions:**
- `getComplianceProfile()`: Fetch profile for current user's org. Return null if none exists.
- `upsertComplianceProfile(formData)`: Parse with complianceProfileSchema. If profile exists, update. If not, insert with org_id from auth. Revalidate /compliance.

**Item actions:**
- `getComplianceItems(filters?: { status?, category?, vehicleId?, driverId? })`: List items with optional filters, ordered by due_date ascending. Include join to get profile's is_dot_regulated for filtering.
- `createComplianceItem(formData)`: Parse with complianceItemSchema. Insert with org_id and compliance_profile_id. Revalidate /compliance.
- `completeComplianceItem(itemId, documentUrls?)`: Set status='completed', completed_date=today. If recurrence_rule exists, call calculateNextDueDate and insert next occurrence (check NOT EXISTS first). Revalidate /compliance.
- `updateComplianceItem(itemId, formData)`: Update item fields. Revalidate /compliance.

**DQ actions:**
- `getDriverQualification(driverId)`: Fetch DQ record for driver. Return null if none. Include computed completeness via calculateDQCompleteness.
- `upsertDriverQualification(driverId, formData)`: Parse with driverQualificationSchema. Upsert by driver_id. Compute and store missing_documents and dq_file_complete. Revalidate /compliance/drivers.
- `getDriversWithDQStatus()`: List all active drivers with their DQ completeness percentage and missing documents count.

**Inspection actions:**
- `createInspection(formData)`: Parse with inspectionSchema. Insert record. Revalidate /compliance/inspections.
- `getInspections(filters?: { vehicleId?, type?, result? })`: List inspections with filters, ordered by inspection_date desc.
- `submitDVIR(formData)`: Parse with dvirSchema. Determine result: 'pass' if all items pass, 'fail' if any fail, 'conditional' if fails exist but corrected. Extract defect descriptions into defects_found array. Create inspection record with inspection_type from form (pre_trip/post_trip). Return created inspection. Revalidate /driver/compliance.

**IFTA actions:**
- `getIFTARecords(quarter?: string)`: List IFTA records for quarter, grouped by jurisdiction. Default to current quarter if not specified.
- `upsertIFTARecord(formData)`: Insert or update IFTA record by vehicle_id + quarter + jurisdiction composite. Revalidate /compliance/ifta.

**Dashboard action:**
- `getComplianceDashboard()`: Fetch all data needed for health score calculation: count of total/overdue/due_soon items, count of drivers with/without complete DQ, count of vehicles with/without current annual inspection. Call calculateHealthScore. Also fetch recent compliance_alerts (last 10 unacknowledged). Return { healthScore, metrics, recentAlerts, upcomingItems (next 90 days ordered by due_date) }.

All actions: get supabase client via createClient(), check auth, scope by org via RLS, return { error?: string } on failure.
