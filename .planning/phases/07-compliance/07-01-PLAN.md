<task type="auto" tdd="true">

  <files>

  <behavior>
    - Health score returns 100 for org with no overdue items, complete DQ files, current inspections
    - Health score returns 0 for org with all items overdue, no DQ files, no inspections
    - Health score applies weights: overdue 40%, due-soon 15%, DQ 25%, inspection 20%
    - DQ completeness returns 100% when all 8 required fields are present and valid
    - DQ completeness identifies expired medical card as missing
    - DQ completeness identifies stale MVR (> 1 year) as missing
    - calculateNextDueDate('2026-01-15', 'annual') returns '2027-01-15'
    - calculateNextDueDate('2026-01-15', 'quarterly') returns '2026-04-15'
    - calculateNextDueDate('2026-01-15', 'monthly') returns '2026-02-15'
    - getDOTRequiredCategories returns full DOT suite; getNonDOTRequiredCategories returns subset
    - IFTA calculateIFTA computes consumed gallons from fleet MPG and jurisdiction miles
    - IFTA exportIFTAToCSV produces valid CSV string with headers
  </behavior>
  <action>
1. Create `src/lib/compliance/compliance-helpers.ts`:
   - `calculateHealthScore(metrics)`: Weighted scoring -- overdue items (40 pts max, penalized 3x ratio), due-soon (15 pts max, penalized 2x ratio), DQ completeness (25 pts max, proportional), inspection currency (20 pts max, proportional). Returns Math.round(sum). Full score when no items/drivers/vehicles exist.
   - `calculateDQCompleteness(dq: DriverQualification)`: Check 8 required fields per FMCSA: application_date, cdl_number, medical_card_expiry (not expired), mvr_last_pulled (within 365 days), road_test_date, annual_review_date (within 365 days), drug_test_last_date, hire_date. Returns { percentage: number, missing: string[] }.
   - `calculateNextDueDate(fromDate, rule, customMonths?)`: Date arithmetic for recurrence. annual +1yr, biennial +2yr, quarterly +3mo, monthly +1mo, custom +N months.
   - `getDOTRequiredCategories()`: Returns full category list for DOT carriers
   - `getNonDOTRequiredCategories()`: Returns subset (insurance, vehicle_registration, state_permit, custom)
   - `isComplianceModuleVisible(category, isDotRegulated)`: Returns boolean for whether a category should be shown for this carrier type

2. Create `src/lib/compliance/ifta-helpers.ts`:
   - `calculateIFTA(records, totalFleetMiles, totalFleetGallons)`: Fleet MPG = totalMiles/totalGallons. Per record: consumed = miles/MPG, netTaxable = consumed - purchased, taxOwed = netTaxable * rate, netTax = taxOwed - taxPaid. Round to 3 decimal places for gallons, 2 for dollars.
   - `exportIFTAToCSV(records)`: Generate CSV with headers: Jurisdiction, Miles, Gallons Purchased, Gallons Consumed, Net Taxable Gallons, Tax Rate, Tax Owed, Tax Paid, Net Tax. Handle proper quoting/escaping.

3. Write `tests/compliance/compliance-helpers.test.ts`:
   - Test calculateHealthScore: perfect score, zero score, partial scores, empty inputs
   - Test calculateDQCompleteness: complete DQ, missing fields, expired medical card, stale MVR
   - Test calculateNextDueDate: all 5 recurrence rules
   - Test getDOTRequiredCategories vs getNonDOTRequiredCategories

4. Write `tests/compliance/ifta-helpers.test.ts`:
   - Test calculateIFTA: basic calculation, zero gallons edge case, multiple jurisdictions
   - Test exportIFTAToCSV: correct headers, data rows, empty input
