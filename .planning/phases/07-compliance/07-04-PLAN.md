1. Create `src/components/compliance/dq-file-tracker.tsx` as client component:
   - Props: drivers (array of driver + DQ + completeness objects)
   - Table/card list of all active drivers showing:
     a) Driver name (first + last)
     b) Completeness percentage as progress bar (green >= 80%, yellow 50-79%, red < 50%)
     c) Missing documents count badge
     d) CDL expiry date with days remaining
     e) Medical card expiry with days remaining
   - Click a driver row to expand/navigate to DQ checklist detail
   - Search by driver name
   - Sort by completeness (ascending = worst first, for action prioritization)

2. Create `src/components/compliance/dq-checklist.tsx` as client component:
   - Props: driverId, dq (DriverQualification | null), completeness
   - Show 8-item checklist (per FMCSA Part 391 from research):
     a) Employment Application (application_date)
     b) CDL Copy (cdl_number)
     c) Medical Certificate (medical_card_expiry -- check not expired)
     d) Motor Vehicle Record (mvr_last_pulled -- check within 365 days)
     e) Road Test Certificate (road_test_date)
     f) Annual Driving Record Review (annual_review_date -- check within 365 days)
     g) Drug Test Result (drug_test_last_date)
     h) Employment Verification (hire_date)
   - Each item shows: icon (check/x/warning), label, status (Present/Missing/Expired/Stale), date if present
   - Missing items highlighted with red text and "Add" button
   - Expired/Stale items highlighted with yellow/orange and "Update" button
   - Inline edit form for updating DQ fields: CDL number/state/class/expiry, medical card expiry, endorsements multi-select, MVR date, drug test date, annual review date, road test date
   - Form uses react-hook-form + zodResolver with driverQualificationSchema
   - Submit calls upsertDriverQualification via useTransition
   - Overall completeness percentage displayed prominently at top

3. Create `src/app/(app)/compliance/drivers/page.tsx` as server component:
   - Call getDriversWithDQStatus()
   - Render page header "Driver Qualification Files"
   - If URL has driverId searchParam, render DQChecklist for that driver
   - Otherwise render DQFileTracker with driver list
