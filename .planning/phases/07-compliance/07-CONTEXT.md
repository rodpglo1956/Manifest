# Phase 7: Compliance - Context

**Gathered:** 2026-03-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Regulatory compliance tracking for DOT and non-DOT carriers. Compliance profiles, trackable compliance items with recurrence, driver qualification files (CDL, medical card, endorsements), vehicle inspections (annual DOT, DVIR pre/post-trip), IFTA quarterly reporting, and compliance health scoring. Adapts based on carrier type (dot_carrier, non_dot_carrier, both) set in org profile.

</domain>

<decisions>
## Implementation Decisions

### Compliance Profile
- compliance_profiles table per PRD-03 Section 2.2: links to org, stores DOT/MC numbers, carrier_type, insurance info, IFTA license, UCR registration, drug testing consortium
- is_dot_regulated boolean determines which compliance modules appear
- carrier_type enum matches organizations.company_type

### Compliance Items
- compliance_items table: individual trackable obligations (DOT inspection, insurance, IFTA, CDL renewal, medical card, etc.)
- Each item has due_date, recurrence_rule ('annual', 'quarterly', 'monthly', 'biennial', 'custom'), alert_days_before array
- Status: 'upcoming', 'due_soon', 'overdue', 'completed', 'waived', 'not_applicable'
- On completion with recurrence, auto-generate next occurrence
- Document uploads linked to compliance items

### Compliance Alerts
- compliance_alerts table: generated by compliance-scanner edge function (pg_cron daily at 6 AM)
- Integrates with proactive_alerts table from Phase 5/6 for Marie visibility
- Alert types: 'approaching', 'due_soon', 'overdue', 'expired', 'completed'
- Push notifications to assigned user and org admins on critical alerts

### Driver Qualification Files
- driver_qualifications table per PRD-03 Section 2.2: CDL info, medical card expiry, endorsements, MVR, drug test, DQ file completeness
- DQ completeness calculated as percentage of required documents present
- Missing documents list for quick identification
- Driver PWA: view own compliance items, upload documents via camera

### Inspections
- inspections table: annual DOT, pre-trip, post-trip, roadside, state, internal
- Result: pass, fail, conditional
- Defects tracking with correction status
- DVIR (Driver Vehicle Inspection Report) form in Driver PWA for pre/post-trip

### IFTA
- ifta_records table: per vehicle, per quarter, per jurisdiction
- Miles traveled, gallons purchased, tax calculations
- Manual entry for Phase 7 (GPS/ELD integration deferred)
- Export to CSV for filing

### Compliance Health Score
- 0-100 score based on: items overdue, items due within 30 days, DQ file completeness, inspection currency
- Displayed on compliance dashboard

### UI Pages
- Command: /compliance (dashboard), /compliance/items (list), /compliance/drivers (DQ tracker), /compliance/ifta, /compliance/inspections
- Driver PWA: /driver/compliance (personal items, DVIR form, document upload)
- Owner-Operator: /oo/compliance (simplified single-carrier view)
- Sidebar: add Compliance section with sub-nav

### Marie Integration
- Marie monitors compliance_alerts and can answer compliance questions
- "When is my IFTA filing due?", "Is my fleet inspection-ready?"
- Already wired via proactive_alerts → Marie panel open hook from Phase 5

### Design Patterns
- All existing patterns from Phases 1-6
- Compliance scanner as SQL function via pg_cron (proven pattern from Phase 4/6)
- StatusBadge extended for compliance status variants
- Document upload reuses FileUpload component from Phase 2

### Claude's Discretion
- DVIR form field layout and grouping
- Compliance dashboard card design
- DQ file checklist visualization
- IFTA data entry form design
- Health score calculation algorithm details

</decisions>

<specifics>
## Specific Ideas

- Compliance should feel like a safety net, not a bureaucratic burden
- The DQ file tracker is critical for DOT carriers — missing one document during an audit is a violation
- DVIR (pre/post-trip inspection) forms need to be fast to fill out — drivers do this every day
- IFTA is quarterly pain for every interstate carrier — making it easy is a differentiator
- Non-DOT carriers (medical transport, local delivery) still need insurance, registration, and vehicle inspection tracking

</specifics>
