  <name>Task 2: CRM pg_cron edge functions</name>
  <files>supabase/migrations/00026_crm_edge_functions.sql</files>
  <action>
Create migration with two pg_cron functions following patterns from 00022_compliance_scanner.sql and 00024_maintenance_monitor.sql:

**1. crm_stats_updater (daily at 2 AM UTC)**

Create function `crm_stats_updater()` that:
- For each org's crm_companies: recalculate total_revenue (SUM of loads.total_charges where loads.broker_name ilike company.name OR loads.pickup_company ilike company.name OR loads.delivery_company ilike company.name, and loads.status in ('delivered','invoiced','paid')), total_loads (COUNT), avg_rate_per_mile (total_revenue / SUM(loads.miles) where miles > 0), last_load_date (MAX of loads.delivery_date or loads.updated_at)
- For each org's crm_lanes: recalculate total_runs (COUNT of loads matching origin city/state and destination city/state), avg_rate_per_mile, last_run_date, last_rate from most recent matching load
- Expire rate agreements: UPDATE crm_rate_agreements SET status='expired' WHERE expiry_date < CURRENT_DATE AND status='active'
- Schedule: `SELECT cron.schedule('crm-stats-updater', '0 2 * * *', 'SELECT crm_stats_updater()')`

**2. follow_up_reminder (daily at 7 AM UTC)**

Create function `follow_up_reminder()` that:
- SELECT crm_activities WHERE follow_up_date <= CURRENT_DATE AND completed_at IS NULL
- For each: INSERT into proactive_alerts (org_id, alert_type='follow_up_due', severity='info', title='Follow-up reminder', message containing activity subject and company name, related_entity_type='crm_activity', related_entity_id=activity.id)
- Use NOT EXISTS with 24-hour window to prevent duplicate alerts (same pattern as compliance scanner)
- For push notifications: INSERT notification data that the existing push notification system can pick up (insert into proactive_alerts which triggers push via existing alert mechanism)
- Schedule: `SELECT cron.schedule('follow-up-reminder', '0 7 * * *', 'SELECT follow_up_reminder()')`

**3. Load completion CRM trigger**

Create trigger function `on_load_delivered()` on loads table AFTER UPDATE:
- When NEW.status = 'delivered' AND OLD.status != 'delivered':
  - Match load to crm_companies by broker_name (ilike match on company name)
  - If match found: UPDATE crm_companies SET total_loads = total_loads + 1, total_revenue = total_revenue + COALESCE(NEW.total_charges, 0), last_load_date = CURRENT_DATE
  - Match load to crm_lanes by pickup_city/state -> destination_city/state
  - If lane match found: UPDATE crm_lanes SET total_runs = total_runs + 1, last_run_date = CURRENT_DATE, last_rate = NEW.total_charges
  - INSERT crm_activities (org_id, activity_type='system', company_id=matched company, subject='Load delivered', body='Load ' || NEW.load_number || ' delivered', user_id=NEW.created_by)

CREATE TRIGGER load_delivered_crm_trigger AFTER UPDATE ON loads FOR EACH ROW WHEN (NEW.status = 'delivered' AND OLD.status IS DISTINCT FROM 'delivered') EXECUTE FUNCTION on_load_delivered();
