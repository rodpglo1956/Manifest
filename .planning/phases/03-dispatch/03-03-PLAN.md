  <action>
    1. Create `src/app/driver/dispatch/page.tsx` (server component):
       - Authenticate user via createClient().auth.getUser()
       - Look up driver record by user_id
       - If no driver record, show "No driver profile linked" message
       - Query current dispatch: from dispatches where driver_id = driver.id AND status NOT IN ('completed', 'rejected'), order by assigned_at desc, limit 1
       - Join load data: load_number, pickup_company, pickup_city, pickup_state, pickup_date, delivery_company, delivery_city, delivery_state, delivery_date, equipment_type, broker_name, rate_amount, rate_type, commodity, weight
       - Get orgId from driver.org_id
       - Pass dispatch (with load data), orgId to DriverDispatchClient

    2. Create `src/app/driver/dispatch/client.tsx` ('use client'):
       - Accept props: dispatch (with joined load), orgId
       - Call useRealtimeDispatches(orgId) for live updates
       - Render DriverDispatchCard with the dispatch data
       - If no active dispatch: show empty state "No active dispatch" with message "You'll see new assignments here when dispatched"

    3. Create `src/components/drivers/driver-dispatch-card.tsx` ('use client'):
       - Big, prominent card (this is the most important thing a driver sees per user's specifics)
       - Card structure:
         a. Header: StatusBadge (variant='dispatch') + "Dispatch" label
         b. Load summary section: load_number, pickup_company + city/state -> delivery_company + city/state, pickup_date, delivery_date, equipment_type, commodity, broker_name
         c. If status === 'assigned': Show prominent Accept (green) and Reject (red) buttons
            - Accept calls acceptDispatch(dispatch.id)
            - Reject shows confirmation ("Are you sure? This will return the load to unassigned.") then calls rejectDispatch(dispatch.id)
         d. If status is 'accepted' or later (not completed/rejected): Show next valid status transitions as buttons
            - Use VALID_DISPATCH_TRANSITIONS[dispatch.status] to get next statuses
            - Each button calls updateDispatchStatus(dispatch.id, nextStatus)
            - Style the primary next action prominently (e.g., "En Route to Pickup" button)
         e. Notes section at bottom: textarea + "Send to Dispatcher" button
            - Calls updateDriverNotes via FormData
            - Shows current driver_notes if any
            - Inline input (not modal) per research recommendation
         f. Show dispatcher_notes if present (read-only, labeled "From Dispatcher")
       - Loading states on all buttons (useTransition or useState for pending)
       - Error display via toast or inline error message
  </action>
