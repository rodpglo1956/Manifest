       - Query loads with status='booked' (unassigned loads), ordered by pickup_date
       - Query active drivers (status='active'), ordered by last_name
       - Query active dispatches (status NOT IN ('completed', 'rejected')), joined with load and driver data. Select dispatch fields + load.load_number, load.pickup_city, load.pickup_state, load.delivery_city, load.delivery_state, load.pickup_date, load.equipment_type + driver first_name, last_name
       - Query all non-terminal dispatches to determine busy driver IDs
       - Query vehicles (status='active') for the assignment form dropdown
       - Pass all data + orgId to DispatchBoard client component

    2. Create `src/app/(app)/dispatch/dispatch-board.tsx` ('use client'):
       - Accept props: unassignedLoads, drivers, activeDispatches, busyDriverIds (Set<string>), vehicles, orgId
       - Call useRealtimeDispatches(orgId) and useRealtimeLoads(orgId) for live updates
       - Render two-column grid: `grid grid-cols-1 lg:grid-cols-2 gap-6`
       - Left: UnassignedLoadsPanel
       - Right: AvailableDriversPanel
       - Below (full width): ActiveDispatchesList
       - Manage selectedLoadId state for the assignment form

    3. Create `src/components/dispatch/unassigned-loads-panel.tsx`:
       - Card with header "Unassigned Loads" and count badge
       - List of load cards showing: load_number, pickup_city/state -> delivery_city/state, pickup_date, equipment_type
       - Each load card has a "Dispatch" button that opens the assignment form (sets selectedLoadId)
       - Empty state: "All loads are dispatched" with link to create new load at /loads/new

    4. Create `src/components/dispatch/available-drivers-panel.tsx`:
       - Card with header "Drivers" and availability count
       - Three sections: "Available" (green badge), "On a Load" (blue badge), "Off" (gray badge - for inactive drivers, but note: only active drivers are queried, so "Off" may be empty unless we include inactive)
       - Per user decision: show active drivers only, categorized as Available (not in busyDriverIds) or On a Load (in busyDriverIds)
       - Each driver card: name, phone, assigned vehicle unit_number (from current_vehicle_id join or passed in), availability badge
       - Compact card row layout (not full table) per research recommendation

    5. Create `src/components/dispatch/dispatch-assignment-form.tsx`:
       - Props: load (the selected load), drivers (available only), vehicles, onClose callback
       - Renders as a section/card when a load is selected (not a modal)
       - Shows selected load summary (load_number, pickup -> delivery)
       - Driver dropdown: select from available drivers (name + vehicle info)
       - Vehicle dropdown: pre-selects driver's current_vehicle_id, allows override from active vehicles list
       - Optional dispatcher_notes textarea
       - Submit calls createDispatch server action via FormData
       - Shows error from server action if any
       - On success: clears selection (load moves to dispatched, disappears from unassigned via Realtime refresh)
       - Uses react-hook-form with zodResolver(createDispatchSchema) per established pattern

    6. Create `src/components/dispatch/active-dispatches-list.tsx`:
       - Card with header "Active Dispatches" and count
       - Table or card list showing: load_number, driver name, pickup -> delivery, StatusBadge (variant='dispatch'), ETA (estimated_pickup_arrival or estimated_delivery_arrival displayed as absolute time, e.g., "ETA: 2:30 PM")
       - If ETA is past, show "(overdue)" label in red
       - Empty state: "No active dispatches"
