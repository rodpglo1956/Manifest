  <action>
1. **Create src/lib/billing/stripe.ts** -- Stripe client singleton:
   - `import Stripe from 'stripe'`
   - Export `stripe` instance using STRIPE_SECRET_KEY env var
   - Use API version '2024-12-18.acacia' (or latest stable)
   - Export helper `getOrCreateStripeCustomer(orgId, orgName, email)`:
     - Query billing_accounts for stripe_customer_id
     - If exists, return it
     - If not, create Stripe customer with metadata.org_id, update billing_accounts, return id
   - Export helper `createCheckoutSession(orgId, plan, billingCycle, successUrl, cancelUrl)`:
     - Gets or creates Stripe customer
     - Looks up price ID from PLAN_CONFIG[plan].stripePriceId[billingCycle]
     - Creates checkout session with subscription mode, org metadata
     - Returns session URL
   - Export helper `createPortalSession(orgId, returnUrl)`:
     - Gets stripe_customer_id from billing_accounts
     - Creates Stripe billing portal session
     - Returns session URL

2. **Create src/app/api/billing/create-checkout/route.ts** -- POST:
   - Authenticate user via createClient(), get org_id from profile
   - Accept body: { plan: BillingPlan, billingCycle: BillingCycle }
   - Validate plan is not 'free' or 'enterprise' (enterprise = contact sales)
   - Call createCheckoutSession()
   - Return { url: session.url }
   - On error return 400/500

3. **Create src/app/api/billing/portal/route.ts** -- POST:
   - Authenticate user, get org_id
   - Verify billing_accounts has stripe_customer_id (402 if no subscription)
   - Call createPortalSession()
   - Return { url: session.url }

4. **Create src/app/api/billing/status/route.ts** -- GET:
   - Authenticate user, get org_id
   - Query billing_accounts for org
   - Query plan_limits for current plan
   - Query usage_records for current period (period_start <= today <= period_end)
   - Return { account: BillingAccount, limits: PlanLimits, usage: UsageRecord | null }

Use supabaseAdmin (from src/lib/supabase/admin.ts) for webhook operations. Use createClient() for user-facing routes.

  <files>src/app/api/billing/webhook/route.ts</files>
  <action>
Create POST handler at /api/billing/webhook:

1. **Signature verification**: Read raw body, verify with STRIPE_WEBHOOK_SECRET via stripe.webhooks.constructEvent(). Return 400 if verification fails.

2. **Event handlers** (use switch on event.type):

   - `customer.subscription.created`: Upsert billing_accounts with plan (from metadata or price lookup), status='active', stripe_subscription_id, current_period_start/end. If trial, set status='trialing'.

   - `customer.subscription.updated`: Update billing_accounts plan, status, billing_cycle, current_period_start/end, payment_method_last4/brand (from default_payment_method if available). Map Stripe status to our status enum.

   - `customer.subscription.deleted`: Update billing_accounts status='canceled', canceled_at=now(), plan='free'.

   - `invoice.paid`: Insert/upsert billing_invoices with stripe_invoice_id, amount (amount_paid/100), tax, total, status='paid', paid_at, pdf_url (invoice.invoice_pdf), period_start/end.

   - `invoice.payment_failed`: Update billing_accounts status='past_due'.

   - `customer.subscription.trial_will_end`: (Optional log/notification -- will be wired in Phase 11 notification system).

3. **Org resolution**: Extract org_id from subscription metadata.org_id. All Stripe objects must include org_id in metadata during checkout creation.

4. **Use supabaseAdmin** (service role) for all database operations since webhook has no user context.

5. **Return 200** on success for all handled events. Return 200 for unhandled events too (Stripe best practice).

6. **Edge runtime config**: Export `const runtime = 'nodejs'` since Stripe SDK needs Node.js. Export `const dynamic = 'force-dynamic'`.

Important: Stripe amounts are in cents. Divide by 100 for database storage.
