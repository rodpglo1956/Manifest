Create migration 00028_billing_tables.sql with these tables per PRD-04 Section 2.2:

1. **billing_accounts** -- one per org:
   - id UUID PK, org_id UUID NOT NULL REFERENCES organizations(id) UNIQUE
   - stripe_customer_id TEXT UNIQUE, stripe_subscription_id TEXT
   - plan TEXT NOT NULL DEFAULT 'free' CHECK (plan IN ('free','starter','professional','enterprise'))
   - billing_cycle TEXT DEFAULT 'monthly' CHECK ('monthly','annual')
   - monthly_rate NUMERIC(10,2), annual_rate NUMERIC(10,2)
   - trial_ends_at TIMESTAMPTZ, current_period_start TIMESTAMPTZ, current_period_end TIMESTAMPTZ
   - status TEXT DEFAULT 'trialing' CHECK ('trialing','active','past_due','canceled','paused','unpaid')
   - payment_method_last4 TEXT, payment_method_brand TEXT
   - cancellation_reason TEXT, canceled_at TIMESTAMPTZ
   - created_at, updated_at with trigger

2. **plan_limits** -- reference table (no RLS, public read):
   - id UUID PK, plan TEXT NOT NULL UNIQUE
   - max_vehicles INTEGER, max_drivers INTEGER, max_loads_per_month INTEGER, max_users INTEGER
   - compliance_module BOOLEAN DEFAULT false, ifta_module BOOLEAN DEFAULT false, crm_module BOOLEAN DEFAULT false
   - ai_assistant BOOLEAN DEFAULT false, ai_queries_per_month INTEGER DEFAULT 0, voice_minutes_per_month INTEGER DEFAULT 0
   - api_access BOOLEAN DEFAULT false, white_label BOOLEAN DEFAULT false, priority_support BOOLEAN DEFAULT false
   - created_at TIMESTAMPTZ DEFAULT now()

3. **usage_records** -- per org per period:
   - id UUID PK, org_id UUID NOT NULL REFERENCES organizations(id)
   - period_start DATE NOT NULL, period_end DATE NOT NULL
   - vehicles_count, drivers_count, loads_count, users_count INTEGER DEFAULT 0
   - ai_queries_count INTEGER DEFAULT 0, voice_minutes_used NUMERIC(8,1) DEFAULT 0
   - created_at, updated_at
   - UNIQUE(org_id, period_start)

4. **billing_invoices** -- cached from Stripe (named billing_invoices to avoid conflict with existing invoices table):
   - id UUID PK, org_id UUID NOT NULL REFERENCES organizations(id)
   - stripe_invoice_id TEXT UNIQUE
   - amount NUMERIC(10,2) NOT NULL, tax NUMERIC(10,2) DEFAULT 0, total NUMERIC(10,2) NOT NULL
   - status TEXT CHECK ('draft','open','paid','void','uncollectible')
   - period_start TIMESTAMPTZ, period_end TIMESTAMPTZ, due_date TIMESTAMPTZ, paid_at TIMESTAMPTZ
   - pdf_url TEXT, created_at TIMESTAMPTZ DEFAULT now()

5. **RLS policies** using auth.org_id() pattern (consistent with project):
   - billing_accounts: org_id = (select auth.org_id())
   - usage_records: org_id = (select auth.org_id())
   - billing_invoices: org_id = (select auth.org_id())
   - plan_limits: GRANT SELECT to authenticated (public read, no RLS needed -- or use permissive policy for all)

6. **Seed plan_limits** with exact values from CONTEXT.md:
   - free: 3 vehicles, 3 drivers, 50 loads/month, 2 users, no modules
   - starter: 10 vehicles, 15 drivers, 200 loads/month, 5 users, compliance + AI (100 queries, 30 voice min)
   - professional: 50 vehicles, 75 drivers, 1000 loads/month, 15 users, all modules (500 AI, 120 voice)
   - enterprise: -1 (unlimited) everything, white_label true, priority_support true

7. **Auto-create billing_accounts** trigger: on INSERT to organizations, create a billing_accounts row with plan='free', status='trialing', trial_ends_at=now()+14 days.

8. **updated_at trigger** on billing_accounts and usage_records (reuse existing trigger function pattern from prior migrations).
