1. **Create src/lib/billing/enforce.ts**:

```typescript
export class UsageLimitError extends Error {
  public statusCode = 402
  public plan: string
  public resource: string
  public current: number
  public limit: number
  constructor(resource, current, limit, plan) { ... }
}

type UsageResource = 'vehicles' | 'drivers' | 'loads' | 'users' | 'ai_queries'

export async function checkUsageLimit(
  orgId: string,
  resource: UsageResource
): Promise<void>
```

Implementation:
- Query billing_accounts for org to get current plan
- If plan is 'enterprise', return immediately (unlimited = -1)
- Query plan_limits for that plan to get the max for the resource
- If max is -1 (unlimited), return
- Count current usage:
  - vehicles: COUNT from vehicles WHERE org_id AND status != 'sold'
  - drivers: COUNT from drivers WHERE org_id AND status = 'active'
  - loads: COUNT from loads WHERE org_id AND created_at >= current period start
  - users: COUNT from org_members WHERE org_id
  - ai_queries: COUNT from marie_queries WHERE org_id AND created_at >= current period start
- Map resource to the correct plan_limits column (vehicles->max_vehicles, etc.)
- If count >= max, throw UsageLimitError with details
- Use supabaseAdmin for these queries (server-side enforcement)

Export helper `getUsageSummary(orgId)` that returns { resource: { current, limit, percentage } } for all resources -- used by billing status UI.

Export helper `checkModuleAccess(orgId, module: 'compliance' | 'ifta' | 'crm' | 'ai')` that checks if the org's plan includes the module. Returns boolean. Does NOT throw -- UI should gate module visibility.

2. **Create supabase/migrations/00029_billing_cron.sql**:

**usage_tracker function** (pg_cron: daily at midnight UTC):
```sql
CREATE OR REPLACE FUNCTION billing_usage_tracker() RETURNS void AS $$
```
- For each org with a billing_account:
  - Count vehicles (status != 'sold'), drivers (status = 'active'), org_members
  - Count loads created in current billing period (current_period_start to current_period_end from billing_accounts)
  - Count ai_queries (marie_queries) in current period
  - UPSERT into usage_records with period_start = current_period_start, period_end = current_period_end
- Schedule: `SELECT cron.schedule('billing-usage-tracker', '0 0 * * *', 'SELECT billing_usage_tracker()')`

**trial_expiry function** (pg_cron: daily at 8 AM UTC):
```sql
CREATE OR REPLACE FUNCTION billing_trial_expiry() RETURNS void AS $$
```
- Find billing_accounts WHERE status = 'trialing' AND trial_ends_at <= now()
- UPDATE plan = 'free', status = 'active', trial_ends_at = NULL
- (Notification will be handled by Phase 11 notification system)
- Schedule: `SELECT cron.schedule('billing-trial-expiry', '0 8 * * *', 'SELECT billing_trial_expiry()')`
