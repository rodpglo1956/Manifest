  <name>Task 1: Maintenance monitor pg_cron and expanded vehicle server actions</name>
  <files>supabase/migrations/00024_maintenance_monitor.sql, src/app/(app)/fleet/actions.ts</files>
  <action>
    1. Create supabase/migrations/00024_maintenance_monitor.sql following the pattern from 00022_compliance_scanner.sql:
       - CREATE OR REPLACE FUNCTION check_maintenance_due() RETURNS void:
         - For each active vehicle, query maintenance_schedules (both vehicle-specific and class-matching where vehicle_id IS NULL)
         - For each schedule, find the most recent maintenance_record of that type for the vehicle
         - Calculate next due: last_service_date + interval_days, OR last_odometer + interval_miles
         - If due within 30 days or 3000 miles: INSERT INTO compliance_items (category='scheduled_service', title='[type] due for [unit_number]', due_date, vehicle_id, status='due_soon') with NOT EXISTS de-duplication (same vehicle_id + maintenance_type + status != 'completed' within last schedule period)
         - If overdue (past due date or past odometer): same insert with status='overdue'
         - Also INSERT INTO proactive_alerts (alert_type='maintenance_due', severity based on priority, related_entity_type='vehicle', related_entity_id) for Marie visibility, with NOT EXISTS 24h de-duplication
       - SELECT cron.schedule('maintenance-monitor', '0 5 * * *', 'SELECT check_maintenance_due()');

    2. Rewrite src/app/(app)/fleet/actions.ts to handle expanded vehicle fields:
       - createVehicle: Accept all new fields from expanded vehicleSchema. Parse with vehicleSchema, insert including vehicle_class, fuel_type, license_plate, license_state, registration_expiry, current_odometer (set odometer_updated_at to now() if provided), avg_mpg, purchase_date, purchase_price, current_value, insurance_policy, notes. Redirect to /fleet.
       - updateVehicle(id, formData): Same expanded fields. If current_odometer changed, update odometer_updated_at to now(). Update single row by id.
       - updateVehicleStatus(id, status): Quick status change action. If status is 'sold' or 'totaled', also unassign current_driver_id (set to null) and close any open vehicle_assignments.
       - deleteVehicle(id): Soft delete by setting status to 'sold'. Do NOT actually delete.
       - Follow existing pattern: 'use server', createClient(), auth check, org check, schema validation, error return format { error: { form: string[] } }.
  </action>

<task type="auto">
  <name>Task 2: Fleet server actions (maintenance, fuel, assignments, cost-per-mile)</name>
  <files>src/lib/fleet/actions.ts</files>
  <action>
    Create src/lib/fleet/actions.ts with 'use server' directive. All actions follow the established pattern: createClient(), auth.getUser(), profile org_id check, schema validation, Supabase query, error handling.

    Maintenance actions:
    - getMaintenanceRecords(vehicleId?: string): Fetch maintenance_records, optionally filtered by vehicle_id, ordered by date_in desc. Return typed array.
    - createMaintenanceRecord(formData: FormData): Parse with maintenanceRecordSchema, insert with org_id. If cost_parts and cost_labor provided, auto-calculate cost_total = cost_parts + cost_labor. If odometer_at_service provided, also update the vehicle's current_odometer and odometer_updated_at. Revalidate /fleet path.
    - updateMaintenanceRecord(id: string, formData: FormData): Update single record. Recalculate cost_total.
    - getUpcomingMaintenance(): Query compliance_items where category='scheduled_service' AND status IN ('upcoming', 'due_soon', 'overdue'), joined with vehicles for unit_number.

    Maintenance schedule actions:
    - getMaintenanceSchedules(vehicleId?: string): Fetch schedules, filter by vehicle_id or return all org schedules.
    - createMaintenanceSchedule(formData: FormData): Parse with maintenanceScheduleSchema, insert with org_id.
    - updateMaintenanceSchedule(id: string, formData: FormData): Update single schedule.
    - deleteMaintenanceSchedule(id: string): Hard delete (schedules are configuration, not records).

    Fuel actions:
    - getFuelTransactions(vehicleId?: string, limit?: number): Fetch fuel_transactions filtered optionally by vehicle_id, ordered by transaction_date desc.
    - createFuelTransaction(formData: FormData): Parse with fuelTransactionSchema, insert with org_id. If odometer_reading provided, update vehicle's current_odometer and odometer_updated_at. After insert, recalculate vehicle avg_mpg from last 10 fuel transactions (using odometer differences between consecutive fill-ups). Revalidate /fleet path.

    Assignment actions:
    - assignVehicle(vehicleId: string, driverId: string, reason?: string): Insert vehicle_assignments record. Update vehicle.current_driver_id. Update driver.current_vehicle_id. Close any existing open assignment for this vehicle (set unassigned_at = now()).
    - unassignVehicle(vehicleId: string, reason?: string): Close current assignment (set unassigned_at). Clear vehicle.current_driver_id. Clear driver.current_vehicle_id.
    - getVehicleAssignmentHistory(vehicleId: string): Fetch vehicle_assignments for vehicle, ordered by assigned_at desc.

    Cost actions:
    - getVehicleCostPerMile(vehicleId: string): Query maintenance_records (sum cost_total), fuel_transactions (sum total_cost), vehicle (purchase_price, current_value, purchase_date, current_odometer) for this vehicle. Use calculateCostPerMile from fleet-helpers. Return { costPerMile, breakdown: { maintenance, fuel, depreciation, insurance, total }, totalMiles }.
    - getFleetCostSummary(): Aggregate across all active vehicles. Return fleet-wide cost per mile, top 5 expensive vehicles, total fleet maintenance and fuel costs.
