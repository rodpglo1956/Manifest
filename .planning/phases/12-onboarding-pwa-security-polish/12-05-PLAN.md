       - syncQueuedActions(): reads queue, replays each action to appropriate server action (dvir -> saveDvirRecord, fuel -> createFuelTransaction), clears on success, keeps on failure
       - registerBackgroundSync(): if 'serviceWorker' in navigator and 'SyncManager' in window, register sync event 'manifest-offline-sync'
       - Called on reconnect (online event) and on app focus
       - Returns {synced: number, failed: number} result

    3. Create src/lib/pwa/use-online-status.ts:
       - useOnlineStatus() hook: tracks navigator.onLine, listens for online/offline events
       - On transition to online: auto-trigger syncQueuedActions()

       - Activate event: clean old cache versions, claim clients
       - Fetch event strategy:
         * For navigation requests: network-first with shell fallback
         * For API requests (/api/*): network-only (no caching API responses to avoid stale data)
         * For static assets (_next/static): cache-first
