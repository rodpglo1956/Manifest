1. Create migration 00031_notifications.sql:

CREATE TABLE notifications:
- id UUID PRIMARY KEY DEFAULT gen_random_uuid()
- org_id UUID NOT NULL REFERENCES organizations(id)
- user_id UUID NOT NULL REFERENCES auth.users(id)
- category TEXT NOT NULL CHECK (category IN ('compliance', 'maintenance', 'load', 'billing', 'crm', 'driver', 'system', 'marie'))
- priority TEXT DEFAULT 'normal' CHECK (priority IN ('low', 'normal', 'high', 'critical'))
- title TEXT NOT NULL
- body TEXT NOT NULL
- action_url TEXT
- action_label TEXT
- read BOOLEAN DEFAULT false
- read_at TIMESTAMPTZ
- channels_sent TEXT[] DEFAULT '{}'
- created_at TIMESTAMPTZ DEFAULT now()
- RLS enabled with user_id = auth.uid() policy
- Index on (user_id, read, created_at DESC) for bell queries

CREATE TABLE notification_preferences:
- id UUID PRIMARY KEY DEFAULT gen_random_uuid()
- user_id UUID NOT NULL REFERENCES auth.users(id) UNIQUE
- compliance_channels TEXT[] DEFAULT '{in_app,push}'
- maintenance_channels TEXT[] DEFAULT '{in_app,push}'
- load_channels TEXT[] DEFAULT '{in_app,push}'
- billing_channels TEXT[] DEFAULT '{in_app,email}'
- crm_channels TEXT[] DEFAULT '{in_app}'
- driver_channels TEXT[] DEFAULT '{in_app,push}'
- system_channels TEXT[] DEFAULT '{in_app}'
- marie_channels TEXT[] DEFAULT '{in_app,push}'
- quiet_hours_start TIME
- quiet_hours_end TIME
- timezone TEXT DEFAULT 'America/New_York'
- created_at TIMESTAMPTZ DEFAULT now()
- updated_at TIMESTAMPTZ DEFAULT now()
- RLS enabled with user_id = auth.uid() policy

Create trigger: auto-create notification_preferences row on profile insert (similar to existing patterns).

  <name>Task 2: Notification dispatcher, actions, and bell UI</name>

1. Create src/lib/notifications/dispatcher.ts:
   - `dispatchNotification(supabase, notification: { org_id, user_id, category, priority, title, body, action_url?, action_label? })`:
     - Inserts into notifications table
     - Loads user's notification_preferences
     - Gets channels for the category (e.g., notification_preferences.load_channels)
     - Checks quiet hours: if current time (in user's timezone) is within quiet_hours_start..quiet_hours_end, skip push/email/sms (in-app still stored)
     - For 'push' channel: call sendPushToUser from existing push lib (fire-and-forget per Phase 6 convention)
     - For 'email' channel: placeholder function `sendEmailNotification()` that logs "Email dispatch not configured" (Resend integration deferred to avoid new dependency complexity)
     - For 'sms' channel: placeholder function `sendSmsNotification()` that logs "SMS dispatch not configured" (Twilio deferred)
     - Updates channels_sent array on the notification record
     - Returns the created notification

2. Create src/lib/notifications/actions.ts with server actions:
   - `getNotifications(limit?: number)`: fetches recent notifications for current user, ordered by created_at DESC, defaults to 20
   - `getUnreadCount()`: returns count of unread notifications for current user
   - `markAsRead(notificationId: string)`: sets read = true, read_at = now()
   - `markAllAsRead()`: marks all unread notifications as read for current user
   - `createNotification(data)`: wraps dispatchNotification for use as server action
   - All use getAuthContext() pattern

3. Create src/components/layout/notification-bell.tsx:
   - Client component with bell icon (lucide-react Bell)
   - Shows red badge with unread count (fetched via getUnreadCount on mount)
   - Subscribes to Supabase Realtime on notifications table (filter: user_id = current user) for live updates
   - Click toggles dropdown panel (positioned absolute, max-h-96, overflow-y-auto)
   - Dropdown shows recent notifications (last 10) with:
     - Category icon/color (compliance=blue, maintenance=orange, load=green, billing=purple, crm=teal, driver=indigo, system=gray, marie=pink)
     - Title (font-medium) and body (text-sm text-gray-500)
     - Time ago (relative: "2m ago", "1h ago", "yesterday")
     - Click navigates to action_url if present, marks as read
   - "Mark all as read" link at top of dropdown
   - Empty state: "No notifications"

4. Update src/components/layout/app-header.tsx:
   - Import and render NotificationBell component in the header, positioned before the user menu/avatar area
