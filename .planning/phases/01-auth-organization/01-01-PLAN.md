    4. `supabase/migrations/00006_auth_helpers.sql` (create BEFORE policies since policies depend on it):
       Create `auth.org_id()` function per Research Pattern 6:
       - language sql, stable, security definer, set search_path = ''
       - Returns: `select org_id from public.profiles where id = auth.uid()`

    5. `supabase/migrations/00004_rls_policies.sql`:
       All policies MUST use `(select auth.org_id())` and `(select auth.uid())` -- the `(select ...)` wrapper is critical for initPlan caching (95% perf improvement).

       Organizations:
       - `org_access`: for all using (id = (select auth.org_id()))
       - `org_insert`: for insert with check (true) -- allow creating new org (no org_id yet)

       Profiles:
       - `users_own_profile`: for all using (id = (select auth.uid())) -- users can always see/update own profile
       - `org_profiles`: for select using (org_id = (select auth.org_id()))

       Org members:
       - `org_members_select`: for select using (org_id = (select auth.org_id()))
       - `org_members_admin_manage`: for insert, update, delete using org_id = (select auth.org_id()) AND exists (select 1 from profiles where id = (select auth.uid()) and role = 'admin')

       Use descriptive policy names per user decision.

    6. `supabase/migrations/00005_auth_trigger.sql`:
       Create `handle_new_user()` trigger function per Research Pattern 8:
       - security definer, set search_path = ''
       - INSERT into profiles with: id = new.id, full_name from metadata (coalesce to ''), role from metadata (coalesce to 'viewer')
       - Also check for org_id and role in raw_user_meta_data (for invitation flow -- if present, set org_id and also insert into org_members)
       - Create trigger on auth.users AFTER INSERT

    7. `supabase/seed.sql`: Create test data for two separate orgs to verify RLS isolation. Each org should have an admin user profile entry. Use comments to explain this is for manual RLS verification.

    **IMPORTANT migration ordering note:** The actual execution order in Supabase is alphabetical by filename. Rename files if needed so that 00006_auth_helpers.sql runs BEFORE 00004_rls_policies.sql, OR combine the auth.org_id() function into the RLS migration file itself (define function at top, then policies below).
