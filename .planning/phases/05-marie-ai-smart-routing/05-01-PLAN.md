    - calculateAvailability returns 1.0 for active driver with no dispatch, 0.0 for driver with active dispatch
    - calculateEquipment returns 1.0 for matching vehicle type, 0.0 for mismatch
    - calculatePerformance returns ratio of on-time deliveries (0.0-1.0)
    - calculateLaneFamiliarity returns normalized count (min(count/5, 1.0))
    - calculateScore with all 1.0 factors returns 100
    - calculateScore with all 0.0 factors returns 0
    - scoreDriversForLoad returns array sorted by score descending
    - scoreDriversForLoad excludes inactive drivers
  </behavior>
  <action>
    1. Create src/lib/routing/adjacency.ts:
       - Export ADJACENT_STATES: Record<string, string[]> mapping each US state abbreviation to its bordering states
       - Export function isAdjacent(state1: string, state2: string): boolean

    2. Create src/lib/routing/factors.ts with individual factor calculators:
       - calculateProximity(driverLastCity, driverLastState, pickupCity, pickupState): number
         Same city+state = 1.0, same state = 0.7, adjacent state = 0.4, far = 0.1
       - calculateAvailability(hasActiveDispatch: boolean): number
         No active dispatch = 1.0, has active = 0.0
       - calculateEquipment(vehicleType: string | null, loadEquipment: string | null): number
         Match = 1.0, no load equipment = 0.8, mismatch = 0.0
       - calculatePerformance(onTimeCount: number, totalDeliveries: number): number
         Ratio clamped 0-1, default 0.5 if no deliveries
       - calculateLaneFamiliarity(previousRunCount: number): number
         min(count / 5, 1.0)
       - All return 0-1 normalized scores

    3. Create src/lib/routing/scoring.ts:
       - Export WEIGHTS = { proximity: 0.30, availability: 0.25, equipment: 0.20, performance: 0.15, lane: 0.10 }
       - Export function calculateScore(factors: ScoringFactors): number -- weighted sum * 100, rounded
       - Export async function scoreDriversForLoad(supabase, load: Load): Promise<DriverSuggestion[]>
         a. Fetch all active drivers with their current_vehicle
         b. For each driver, get last completed dispatch -> load delivery city/state (for proximity)
         c. Check for active dispatch (for availability)
         d. Match vehicle_type vs load equipment_type (for equipment)
         e. Count on-time deliveries from load_status_history (for performance)
         f. Count previous loads on same origin-destination city pair (for lane familiarity)
         g. Calculate weighted score, sort descending, return top 10

    4. Create src/app/api/dispatch/suggest/route.ts:
