    Create server actions file at src/app/(app)/drivers/actions.ts with 'use server' directive:
    - createDriver(formData: FormData): validates with driverSchema.safeParse, inserts into drivers table via Supabase server client, redirects to /drivers on success, returns { error: { form: string[] } } on failure
    - updateDriver(id: string, formData: FormData): validates with driverSchema.safeParse, updates drivers record by id, redirects to /drivers/[id] on success
    - deactivateDriver(id: string, status: 'inactive' | 'terminated'): updates driver status, revalidates /drivers path

    Follow exact pattern from src/app/(auth)/signup/actions.ts: create Supabase server client with `await createClient()`, use safeParse for validation, return error objects on failure, use redirect() on success.

    Create reusable StatusBadge component at src/components/ui/status-badge.tsx:
    - Props: status (string), variant (optional: 'driver' | 'vehicle' | 'load')
    - Maps status to Tailwind color classes (active=green, inactive=yellow, terminated=red for drivers; active=green, inactive=gray for vehicles)
    - Renders as a small pill/badge with colored dot + text
    - Uses design system constraints: rounded-md (6px), no heavy shadows
  </action>
