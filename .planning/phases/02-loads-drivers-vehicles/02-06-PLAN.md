  <action>
    Create driver server actions at src/app/driver/loads/actions.ts:
    - driverUpdateStatus(loadId: string, newStatus: LoadStatus): validates transition, updates load status. Same as updateLoadStatus but scoped -- verifies the current user's driver record is assigned to this load before allowing update.
    - driverUploadDocument(loadId: string, docType: 'bol' | 'pod', formData: FormData): extracts file from FormData, calls uploadLoadDocument, updates load's bol_url or pod_url. Only BOL and POD (not rate_confirmation) for drivers.

    Create driver-active-load.tsx ('use client'): the prominent active load card for Driver PWA.
    - Receives load (Load with joined vehicle) and orgId as props
    - Large card taking most of the screen width
    - Shows: load_number (font-mono, large), pickup city -> delivery city (large text), pickup date and time window, delivery date and time window, equipment type, vehicle unit# if assigned
    - Status update buttons: big, full-width, thumb-friendly (min-height 48px). Show only the NEXT valid status transitions. For example, if status is 'in_transit', show "At Pickup" button. If 'loaded', show "At Delivery". Use getStatusLabel for button text.
    - Each button calls driverUpdateStatus server action
    - Below buttons: two camera upload areas for BOL and POD using FileUpload with isMobile=true
    - Design: large text, high contrast, big touch targets. This is used from a truck cab.

    Create driver-load-history.tsx: receives loads array (past 30 days). Simple list of load cards showing load_number, pickup->delivery, date, status badge. Links to /driver/loads/[id] for detail.

    Create /driver/loads/page.tsx: server component.
    - Queries the current user's driver record (drivers table where user_id = auth.uid())
    - Queries active load (loads where driver_id = driver.id AND status in active statuses: dispatched, in_transit, at_pickup, loaded, at_delivery)
    - Queries load history (loads where driver_id = driver.id AND created_at > 30 days ago, ordered by created_at desc)
    - If active load exists: render driver-active-load prominently
    - If no active load: show "No active load" empty state
    - Below: render driver-load-history
    - Wrap in client component using useRealtimeLoads for live updates

    Create /driver/loads/[id]/page.tsx: server component, queries load by id (verifying driver assignment), shows simplified load detail (pickup, delivery, status, documents).

    Update src/components/layout/driver-header.tsx: add "Loads" and "Settings" to the bottom navigation bar. Loads icon: Package (lucide-react). Settings icon: Settings. Dashboard remains. Loads href: /driver/loads. Settings href: /driver/settings.

    Create tests/driver/loads.test.ts with todo stubs for LOAD-15, LOAD-16.
