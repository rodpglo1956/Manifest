```typescript
export const pickupSchema = z.object({ pickup_company: z.string().min(1), ... })
export const deliverySchema = z.object({ delivery_company: z.string().min(1), ... })
export const freightSchema = z.object({ commodity: z.string().optional(), ... })
export const rateSchema = z.object({ rate: z.coerce.number().positive(), ... })
export const brokerSchema = z.object({ broker_name: z.string().optional(), ... })
export const assignmentSchema = z.object({ driver_id: z.string().uuid().optional(), ... })
export const loadSchema = pickupSchema.merge(deliverySchema).merge(freightSchema).merge(rateSchema).merge(assignmentSchema).merge(brokerSchema)
export const STEP_FIELDS = { pickup: [...], delivery: [...], freight: [...], rate: [...], assignment: [...] }
export type LoadInput = z.input<typeof loadSchema>
```

    Create load server action at src/app/(app)/loads/actions.ts:
    - createLoad(formData: FormData): validates all fields with loadSchema.safeParse, computes total_revenue (rate + fuel_surcharge + accessorial_charges), inserts into loads table with created_by = current user id, redirects to /loads on success. Load number is NOT set (database trigger auto-generates it).

    Create load-wizard.tsx ('use client'): the wizard container per research Pattern 1.
    - Single useForm<LoadInput>() instance with zodResolver(loadSchema)
    - 5 steps: Pickup, Delivery, Freight, Rate & Broker, Assignment
    - Progress bar at top showing current step with colored indicators
    - handleNext() validates current step fields via methods.trigger(STEP_FIELDS[stepKey])
    - handleBack() decrements step (no validation needed)
    - Final step submits via handleSubmit -> converts to FormData -> calls createLoad server action
    - Wrap children in FormProvider so step components access form context
    - Step components use useFormContext() to access register/errors

    Create step components (all 'use client'):
    - step-pickup.tsx: pickup_company, pickup_address, pickup_city, pickup_state (2-char), pickup_zip, pickup_date (date input), pickup_time_start (time input, optional), pickup_time_end (time input, optional), pickup_reference (optional)
    - step-delivery.tsx: same pattern as pickup but delivery_ prefixed fields
    - step-freight.tsx: commodity (optional), weight (number, optional), weight_unit (select: lbs/kg), pieces (number, optional), equipment_type (select: 6 types), temperature_min/max (number, optional, only shown when equipment_type is 'reefer'), hazmat (checkbox)
    - step-rate.tsx: rate (number, required), rate_type (select: flat/per_mile/hourly/per_stop), estimated_miles (number, optional), fuel_surcharge (number, default 0), accessorial_charges (number, default 0), broker_name, broker_mc (font-mono), broker_contact, broker_phone, broker_email. Show computed total at bottom.
    - step-assignment.tsx: driver_id (select from org drivers, optional), vehicle_id (select from org vehicles, optional). Fetch available drivers and vehicles. Show "Assignment is optional. You can assign later from Dispatch." note. This step also shows notes textarea.
