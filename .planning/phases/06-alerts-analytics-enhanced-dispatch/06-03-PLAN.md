    - After successful dispatch creation, check driver's notification preferences
    - If new_dispatch preference is enabled, call sendPushToUser for the driver's user_id with payload: { title: 'New Dispatch', body: `Load ${loadNumber} assigned to you`, url: '/driver/dispatch', tag: 'dispatch-' + dispatchId }
    - Look up driver's user_id from drivers table
    - Import sendPushToUser from '@/lib/push/send-notification'
    - Wrap push sending in try/catch -- push failure should NOT fail the dispatch creation (fire-and-forget pattern)
    - Also wire conflict detection: import checkDispatchConflict from Plan 01, call before creating dispatch. If conflict, insert proactive_alert with type 'dispatch_conflict' and severity 'critical', then send push to admins/dispatchers. Still allow the dispatch (warn, don't block -- per ALRT-04 which says "alert", not "prevent").

    **src/app/(app)/loads/status-actions.ts -- extend updateLoadStatus:**
    - After successful status update, find the dispatcher who created the dispatch for this load
    - If load_status_change preference is enabled for that dispatcher, call sendPushToUser with payload: { title: 'Load Status Update', body: `Load ${loadNumber} is now ${newStatus}`, url: `/loads/${loadId}`, tag: 'status-' + loadId }
    - Wrap in try/catch (fire-and-forget)

    **src/lib/alerts/actions.ts -- extend with push for critical alerts:**
    - Add a helper triggerAlertPush(supabase, alert: ProactiveAlert) that checks if alert.severity === 'critical' and sends push to org admins/dispatchers
    - This function will be called by a Realtime listener or can be added as a post-insert pattern. For simplicity in Phase 6, add a Supabase database webhook or trigger it from the dashboard Realtime subscription when new critical alerts arrive.
    - Alternative (simpler): In the alert feed component, when new critical alerts arrive via Realtime, trigger a client-side notification if permission is granted (using the Notification API directly). This avoids the pg_net complexity and still satisfies PUSH-04.
