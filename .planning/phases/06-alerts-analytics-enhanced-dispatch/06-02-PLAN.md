<task type="auto" tdd="true">
  <name>Task 1: Service worker, VAPID config, push subscription API, and send utility</name>
  <files>public/sw.js, src/lib/push/vapid.ts, src/lib/push/send-notification.ts, src/app/api/push/subscribe/route.ts, tests/push/subscribe.test.ts</files>
  <behavior>
    - Service worker push handler extracts title, body, url from event data
    - Service worker notificationclick opens the URL from notification data
    - VAPID config reads from env vars NEXT_PUBLIC_VAPID_PUBLIC_KEY and VAPID_PRIVATE_KEY
    - Subscribe endpoint saves subscription to push_subscriptions table
    - Subscribe DELETE endpoint removes subscription by endpoint
    - sendPushToUser queries subscriptions for a user and sends via web-push
    - sendPushToUser cleans up expired subscriptions (410 Gone)
  </behavior>
  <action>
    **public/sw.js:**
    - self.addEventListener('push') -- parse event.data?.json(), show notification with title, body, icon (/icons/icon-192.png), badge, tag, data.url
    - self.addEventListener('notificationclick') -- close notification, clients.openWindow(notification.data?.url || '/')
    - self.addEventListener('install') -- self.skipWaiting()
    - self.addEventListener('activate') -- clients.claim()

    **src/lib/push/vapid.ts:**
    - Export VAPID_PUBLIC_KEY = process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY
    - Export VAPID_PRIVATE_KEY = process.env.VAPID_PRIVATE_KEY
    - Export VAPID_SUBJECT = 'mailto:support@glomatrix.com'

    **src/lib/push/send-notification.ts:**
    - Import webpush from 'web-push', set VAPID details using vapid.ts exports
    - Export sendPushToUser(supabase, userId, payload: { title, body, url?, tag? }): Promise<void> -- query push_subscriptions for user_id, send to all endpoints, delete 410 Gone subscriptions
    - Export sendPushToRole(supabase, orgId, role: UserRole, payload): Promise<void> -- query profiles by org_id + role, then send to each user
    - Export sendPushToOrgAdminsAndDispatchers(supabase, orgId, payload): Promise<void> -- convenience wrapper for critical alerts

    **src/app/api/push/subscribe/route.ts:**
    - POST: authenticate user, read body { endpoint, keys: { p256dh, auth } }, insert into push_subscriptions (user_id from session, org_id from profile), return 201. Use ON CONFLICT (user_id, endpoint) DO UPDATE for re-subscription.
    - DELETE: authenticate user, read body { endpoint }, delete from push_subscriptions where user_id + endpoint match, return 200.

    **tests/push/subscribe.test.ts:**
    - Test that sendPushToUser handles empty subscription list gracefully
    - Test that sendPushToUser processes multiple subscriptions
    - Test 410 cleanup logic (mock webpush.sendNotification rejecting with statusCode 410)
  </action>
